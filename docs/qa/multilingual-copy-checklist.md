# Multilingual Copy QA Checklist

Use this checklist for every onboarding, assistant or tax-sensitive copy change.

## Scope

- Languages in current MVP: English and Turkish.
- First reviewed flow: onboarding language selection and setup screens.
- Official English tax terms must remain visible when the user preference is enabled.

## Required Checks

### Meaning and Safety

- [ ] English and Turkish copy communicate the same user outcome.
- [ ] Copy avoids final tax advice unless reviewed by a qualified professional.
- [ ] Estimates, assumptions and uncertainty use cautious language.
- [ ] Accountant review prompts remain clear in both languages.
- [ ] Official terms such as `allowable expense`, `Self Assessment` and `tax year` are preserved in English where needed.

### UI Fit

- [ ] Copy fits on small mobile screens without truncating important meaning.
- [ ] Turkish text expansion does not overlap buttons, cards or navigation headers.
- [ ] Buttons remain short enough for 320px-wide screens.
- [ ] Dynamic font scaling keeps primary actions reachable.

### Preference Behaviour

- [ ] Selected app language persists after app restart.
- [ ] Official English tax terms visibility preference persists after app restart.
- [ ] Toggling language does not reset setup progress.
- [ ] The user can understand what the official tax-term preference changes.

### Accessibility

- [ ] Button labels are understandable without surrounding visual context.
- [ ] Header text describes the current step.
- [ ] Colour is not the only indicator of selected language or tax-term preference.
- [ ] Screen reader order follows title, body, options and action.
